# 보안 점검 에이전트 정책 — 기본 (policies/CLAUDE_base.md)

이 파일은 '정책 전/후 비교' 실습의 '전(before)' 상태입니다.
기본 점검 원칙만 담고, 차량 제어 모듈 심각도 가중 규칙은 포함하지 않습니다.

## 점검 원칙
- 모든 발견에는 CWE 식별자와 근거 코드를 반드시 포함한다.
- 비밀 값은 어떤 출력에서도 원문을 표시하지 않는다(마스킹 필수).
- 점검하지 못한 영역은 '미점검'으로 명시한다(침묵하지 않는다).

## 데이터 보호
- 차량 SW 소스·ECU 설정·보안 아키텍처 문서는 외부 전송 통제 대상이다.
- 점검 대상은 교육용 sample_app으로 한정한다(운영·외부 자산 점검 금지).
